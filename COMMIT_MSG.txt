Show accounting menu items as synthetic entries

The Primanota/Storico/Bilancio/Partitari items did not appear in the sidebar:
they were only mapped via FORM_TO_URL, which works only if the legacy tbl_menu
has rows pointing at those VFP forms. Those rows aren't present, so nothing was
translated and no item showed.

Inject them as synthetic entries under the "contabilita" top-level, the same
mechanism already used for magazzino/parametri: Primanota, Storico contabile,
Bilancio, Partitario clienti, Partitario fornitori, then the existing chart-of-
accounts shortcuts. hasSyntheticEntries already covers "contabilita", so the
top-level stays visible even without reachable legacy L2 items. The FORM_TO_URL
accounting entries are kept as a harmless fallback.

Rebased on top of the supplier and accounting patches (touches MenuService
only). Verification: brace/paren scan, git apply --check clean on a fresh
clone with both patches applied. Menu cache may need eviction/restart to show.
