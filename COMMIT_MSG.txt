Hotfix legami viewer: empty node detail (malformed inline SVG)

Opening a node detail (e.g. /utilita/legami?id=prg:aq_browseview) showed an empty
card (only header + search). Cause: neighborhoodSvg() built clickable-node hrefs
with an unescaped '&' ("?id=...&trail=..."), making the inline SVG invalid XML;
the browser's foreign-content parser then swallowed the rest of the node content.

Fix:
- SVG hrefs now use '&amp;trail=' (valid XML; generated SVG verified well-formed).
- The id branch returns only when the node exists; an unknown id now degrades to
  the grid with an "object not found" notice instead of falling through to the grid
  with unset pagination variables.

Version 0.23.1. Sandbox syntactic checks only (no Maven build).
