Fix empty node detail in legami viewer (restore th:with pattern)

Removing the SVG (0.23.2) did not fix the empty node detail: it was empty for ALL
nodes (PRG and MENU). Real cause in the template: the 0.23.0 rewrite changed the
parents/children rendering from the working Fable5 pattern
  th:each="pe : ${parents}" th:with="l=${pe[0]}, o=${pe[1]}"  ... ${o.id}
to direct Object[] indexing with property access plus a multi-param link expression
with a space
  th:each="p : ${parents}" ... ${p[1].id} ... @{...(id=${p[1].id}, trail=...)}
That combination broke rendering of the whole node block.

Fix: restore the th:with extraction (l/o and cl/co) for parents and children and
drop the space after the comma in the link expressions
(@{/utilita/legami(id=${o.id},trail=${trailNext})}). The grid (r[0].*) is unchanged
- it uses a single, comma-less parameter and works.

All grid improvements remain (full data, counter filters, pagination, inline search,
menu-orphan fix, form titles). The neighborhood graph will return as an isolated
endpoint.

Version 0.23.3. Sandbox syntactic checks only (no Maven build).
