Gate unified prima nota menu entry on legacy UTENTI permission

Collapsing the "Prima nota" submenu into one entry dropped the legacy
authorization: the entry showed for anyone who could see the Contabilità
menu, including users not enabled for any prima nota item (tbl_menu filters
per leaf by the UTENTI field, DB-side).

In the container transformation, `children` holds only the leaves the user is
allowed to see. The unified entry is now produced only when at least one
non-separator leaf is visible; otherwise the container returns null (the
caller already handles that). A user with no prima nota permission no longer
sees the entry.

Per-action granularity (READ/CREATE/UPDATE/DELETE/LOCK with conditional
buttons and server-side enforcement) is deferred to the write CRUD slice,
when those actions actually exist; today the entry opens the read-only
consultation, matching the legacy search/consult permission.

Rebased on the supplier, accounting and prima-nota-menu patches; git apply
--check clean. Menu cache needs a restart to refresh.
