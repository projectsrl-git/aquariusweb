Fix accounting menu code (contabilit) so synthetic entries appear

The accounting L1 menu code in tbl_menu is literally "contabilit", not
"contabilita" (not a truncation — the column is varchar(50)). guessIcon already
handled both spellings, but injectSyntheticEntries and hasSyntheticEntries only
matched "contabilita", so the section rendered (icon + primanota) while the
web-only entries — Storico contabile, Bilancio, Partitario clienti/fornitori,
Piano dei conti — were silently never injected.

Aligned both methods to match "contabilit" as well. Documented the gotcha in
CLAUDE.md. No behavior change for magazzino/parametri.

Sandbox checks only (no Maven). After deploy (Tomcat restart clears the menu
cache) the four entries should appear active under Contabilità.
