Fix duplicate column mappings that prevented startup

The customer-tabs change mapped five columns twice, so Hibernate refused to
build the tenant EntityManagerFactory (Repeated column in mapping ...
cli_impfid) and the app failed to start.

Five columns were already mapped by the original tabs and got re-added for
the new ones: CLI_IMPFID, CLI_CLFIDO, CLI_EMAIL, CLI_EMAIL1, CLI_EMAIL2.

Removed the duplicate (new) fields from Customer and their setters, and
remapped the affected tabs to the existing properties: Fido -> creditLimit /
creditLimitEnabled, Gruppi E-mail -> email / emailAlt1 / emailAlt2. In the
VFP form these tabs are just other views of the same columns.

Verification: no duplicate @Column remains (added this scan to the review),
Java brace/paren scan, every th:field maps to a property, edit.html div
balance 232/232, git apply --check clean on a fresh clone of HEAD e581084.
