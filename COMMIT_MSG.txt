Fix sidebar menu links missing the context path

Clicking a dynamic menu item (e.g. Magazzino > Valorizzazione) navigated to
/magazzino/valorizzazione and returned 404 instead of
/aquariusweb/magazzino/valorizzazione.

The sidebar is rendered in JS from /api/menu/tree, where each node.url is an
absolute server path. renderNode() assigned item.href = node.url verbatim, so
under the Tomcat context the links pointed at the server root. The context
helper existed but was scoped inside loadMenu() and invisible to renderNode().

Hoist it to script scope as MENU_CTX and prefix the href:
item.href = MENU_CTX + node.url. The menu fetch keeps using it too. The chart
of accounts tree already builds hrefs from @{/conti/}, so it was unaffected.

Verification: node --check on the menu script, no remaining JS absolute
hrefs without context, git apply --check clean on a fresh clone of HEAD
42d572b. Navigation to confirm on deploy.
