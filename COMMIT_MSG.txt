Add program objects catalog and link graph CSVs

Part 1 of 2. Object catalog + parent->child link graph of the legacy
codebase, menu-reachable closure (read-only source analysis, no
execution). Applies on HEAD 64cf132.

PERIMETER: 885 forms referenced by tbl_menu (Caption + METHODS memos
parsed), forms they open, 366 called PRGs - 123 of which are LIBRARIES
decomposed into 975 PROCEDURE/FUNCTION nodes (id prg:<file>::<NAME>,
description = adjacent comment), FRX print targets, 1050 menu-voce
nodes (type MENU - declared deviation from the mandate enum: the
viewer needs labelled menu parents), 62 WEB endpoint nodes.
TOTALS: 3925 objects, 5618 links.

LINK TYPES: MENU_TO_FORM (1007), OPENS_FORM (1623, DO FORM),
CALLS_PRG (1570, DO x + SET PROCEDURE/LIBRARY), CALLS_PROC (1315:
direct DO resolved to unique library procedures + name() calls with
name >= 5 chars to avoid false positives), PRINTS_REPORT (32),
MIGRATED_TO (64, from tracker new_location + audit evidenza_web),
UNRESOLVED (7, &macro/EXECSCRIPT - never invented). ~603 DO targets
matching neither a disk PRG nor a unique library procedure were NOT
forced into links (local form procedures or out-of-closure): counted
and explained in migration/README.md.

FILES: migration/program_objects.csv (object_id, file, object_type,
name, description, new_location - RFC-4180), program_links.csv
(parent_id, child_id, link_type, evidence), README.md updated with
the schema, link types and library-decomposition logic.
