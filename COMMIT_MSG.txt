Add magazzino, distinta base, produzione standard (read-only)

Session-2 horizontal expansion, three consultation modules in one
cumulative patch (they share pom.xml, CLAUDE.md, migration tracker and
MenuService). Version bump 0.2.0 -> 0.3.0 (MINOR, new user-visible
modules; log line added in CLAUDE.md 4b).

MAGAZZINO (/magazzino/movimenti, /magazzino/giacenze)
- WarehouseMovement (U_MAG_MO): fiscal-year list, TOP causale badge
  decoded via PARA 'TOP'+MOV_TOP -> DESCRI (verbatim from the
  menu_movimenti_mag query; same TOP rule as primanota). Dates stay
  strings (dirty MOV_DTREGI, never converted).
- StockBalanceDao (U_MAG_GG, native SQL on tenantDataSource): the legacy
  aggregation SUM(MAG_GIACEN) GROUP BY MAG_ANAART, MAG_CODMAG - counting
  groups with HAVING needs a derived table, not expressible in
  JPQL/Hibernate 5. Anomalies surfaced, not hidden: badge "Negativa" and
  "Zero con storico" (phantom rows); switch to include zero balances.
- Menu: menu_movimenti_mag via FORM_TO_URL; giacenze via NEW minimal
  FUNCTION_TO_URL mechanism (commands '=funzione()') mapping
  determina_form_giacenze -> /magazzino/giacenze (APPLILIB dispatcher,
  non-medical installs open MENU_GIACENZE).

DISTINTA BASE (/distinte)
- BomHead/BomRow (U_DIS_TT/DD): DIT_GRUPPO = parent ARTICLE code (join
  U_ART_PR.ART_CODPRI in DISTINTA_BASE_LIB); first-level components only;
  DIS_ESPLOD='X' -> "Apri sotto-distinta" link, one level at a time via
  /distinte/articolo/{code}. Multi-level explosion and costing DEFERRED.
- Menu: menu_dis000 -> /distinte.

PRODUZIONE STANDARD (/produzione)
- STANDARD discriminator VERIFIED in STD_PROGRAMMAZIONE sources:
  PARENT = '' AND TIPO = 'STD' on PRODUZIONE (root nodes of the program
  tree). PROD_ORDINI/PROD_LEGAMI/PROD_AVANZA linked by IDPRG (no soc/year
  columns). GRUPRD badge decoded via PARA 'PRD'+code (legacy seek_para).
- Detail: program header (planned vs actual), linked customer orders,
  components (PRDFIN badge), progress planned-vs-produced. Non-standard
  production variants excluded by mandate (OUT_OF_SCOPE in tracker);
  std_avanza is data-entry and intentionally NOT menu-mapped.

Migration tracker: +11 rows (menu_movimenti_mag, menu_giacenze,
MENU_DIS000, STD_PROGRAMMAZIONE, STD_AVANZA). Three slice docs in
.claude/ + three rows in the CLAUDE.md slice table.
