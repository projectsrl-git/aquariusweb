Add read-only customer orders module (/ordini)

New consultation module for U_ORD_TT + U_ORD_DD, web counterpart of VFP
MENU_ORD000 ("Ordini clienti"). Depends on the articles patch (applies
after it: shared edits to MenuService/CLAUDE.md/migration tracker).

- OrderHead/OrderRow read-only entities; list scoped to society + fiscal
  year via FiscalContext (legacy PUB_ANNO behaviour).
- Header-rows join: primary hook TT.TAGGANCIO = DD.DAGGANCIO (the
  "aggancio" key used by modern legacy SQL joins on ALL document tables);
  fallback triple key ORS_DATORD/ORS_NUMORD/ORS_CODCLI (ristampelib) for
  old records with empty hook.
- Row semantics from APPLILIB: ORD_VALORE = ORD_PRZNET x ORD_QTAORD;
  comment rows ('*** COMMENTO ***') rendered muted.
- Status badge from ORD_CHIUSO/ORD_EVATOT/ORD_EVAPAR
  (Chiuso/Evaso/Evaso parz./Aperto).
- MenuService FORM_TO_URL: menu_ord000 -> /ordini.
- Migration tracker: 14 rows for MENU_ORD000.SCX.

Verification (sandbox only): brace balance OK, duplicate @Column empty,
fragment arity OK, CSV OK. Applies cleanly after the articles patch
(chain verified on fresh clone of HEAD d4652ec). Maven build, live-DB
behaviour and LIKE-search performance on large archives to be confirmed
at first deploy.
