Add Maven environment profiles and per-env secrets layout

Select the target environment at build time with a Maven profile
(-Pdev default, -Psit, -Puat, -Pprod). Each profile is filtered into
application.properties (@env.name@ -> spring.profiles.active,
@aquariusweb.home@ -> home dir) via resource filtering.

Secrets never enter git. application.properties imports, at runtime,
config/application-<active-profile>.properties, which is gitignored.
A single versioned template (config/application-ENV.properties.template)
documents the keys; copy it once per environment and fill in real URLs
and credentials. Non-sensitive per-env overrides (Thymeleaf cache, log
levels) live in versioned application-<env>.properties inside resources.

.gitignore now ignores config/application-*.properties while keeping the
template, plus H2 runtime db files. Also removes the dead recursive tree
fragment that lingered after the aq-tree.js rework.

Verification: only the placeholder template is staged (no real hosts or
credentials); git apply --check clean on a fresh clone of HEAD 5cf893a.
Maven filtering and Tomcat 9 run to be confirmed on target.
