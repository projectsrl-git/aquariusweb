Add CONTABILELIB inventory (mapping for primanota posting)

Map the VFP accounting library (contabilelib.PRG, ~59,900 lines, 422
procedures) before replicating its primanota logic in the web system. This
is documentation only: no code, schema, or logic changes.

The document inventories the key posting procedures with their sizes
(REGISTRA 793 lines, OK_REG, AUT_CONTROPARTITE, REG_ANCLFO_PART_CLI/FOR,
REG_DETIVA, CALCOLO_IMPOSTA, COD_PAGAM, QUA_REG, REG_NUMRPN), the causale
matrix (TAB_CAUCONT flags that drive REGISTRA, with inferred semantics to be
confirmed with Erasmo), the special cases already layered in the code
(cespiti, supplier/customer advances, reverse-charge self-invoices,
compensations), and a proposed replication order from the simplest case
(pure giroconto) up, each with web-vs-VFP reconciliation on a test DB.

The goal is an explicit map to plan the posting slices and to review with
Erasmo, not a presumption of completeness. REGISTRA is heavily stratified
(annotations dated through 2024) and must be replicated one case at a time.
