Add CEE balance DEEP analysis: structure, mapping, algorithm

Sources-verified decode of the legacy "Bilancio IV direttiva CEE" flow so
Opus can implement the calculation engine on certain ground. No code
changes: catalogs + README + tracker.

KEY FINDINGS (all cited file/line in the deliverables)
- BILNEW holds structure AND values: of 39 columns the CEE flow uses six
  (CODSOC, CODRIG, TIPO_DATO, DESCRIZION, CORRENTE computed/reset each
  run, PRECEDENTE rolled by menu_ceeanno: PRECEDENTE=CORRENTE;CORRENTE=0).
  The other 33 are never read by the flow (generic-table legacy, INCERTO).
- NO parent-child tree in BILNEW. Structure = print order BIL_CODRIG
  (numeric-in-varchar, PADL-10 comparisons) + row type TIPO_DATO with
  verbatim validation "I=Commenti, V=Dettagli di riga, T=Totali"
  (menu_cee000; STBILCEE.FRX styles T rows) + totals hierarchy stored in
  U_COR_TT as a signed edge list riga->totale ('+' adds, else subtracts;
  missing CONFLU row ABORTS the run).
- BIL_CODRIG number space encodes the section: VAL(codrig) >= 21600 =
  conto economico (ABS at end of phase 2), < 21600 = stato patrimoniale.
  Attivo/passivo boundary is data, not code (INCERTO for Opus).
- U_INT_TT dual destination: INT_CODRIG "Codice riga dare" (saldo >= 0),
  INT_CODRIA "Codice riga avere" used when the saldo is NEGATIVE if
  filled (form help verbatim; source comment "BANCHE IN PASSIVO").
- Full ceecont.PRG pseudocode in resources/cee/README.md, with the
  previsionali option (PRE_IMP_D-PRE_IMP_A), the three data-quality
  behaviors (unmapped accounts SILENTLY excluded; mappings to missing
  rows skip the account with verbatim warning; broken total edges abort)
  and ready SSMS queries. Commented-out cases 21900/24400 ("da vedere col
  commercialista") flagged INCERTO.
