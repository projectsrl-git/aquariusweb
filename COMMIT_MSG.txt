Add sales revenue statistics with multi-year comparison

Patch A of session 10. /vendite/statistiche built ON TOP of the
existing invoice tables (no document list rebuilt): web equivalent of
menu_fatturato / "Analisi fatturato su piu anni". Version bump
0.18.1 -> 0.19.0. Applies on HEAD bb97ba5.

DATA: VenditeStatsDao (native, tenantDataSource) on U_FAT_TT heads
(measure = ORD_IMPONIB alongside t_netto) and U_FAT_DD rows (per
article: ORD_QTAORD + ORD_VALORE = row value net of discounts; head-row
join TAGGANCIO=DAGGANCIO as in InvoiceRowRepository). Month derived
from ORD_DATORD 'yyyy/MM/dd' SUBSTRING(6,2); multi-year scan guarded
with TRY_CONVERT(INT, ORD_ANNO).

VIEWS: KPI cards (fatture, imponibile totale); per month (with optional
month-range filter); multi-year anno x mese matrix (last 5 years) +
totals per year; per customer (top 100 by imponibile); per article
(top 100 by row value). Excel export via VenditeStatsExcelExporter
(SXSSF pattern from BilancioExcelExporter): sheets Per mese /
Per cliente / Per articolo.

HONESTY: note di accredito are NOT netted nor excluded (no sign
assumptions - counts and amounts as stored); the per-article measure
is labelled "valore righe"; legacy filters causale/zona/operatore/
c.costo not reproduced (tracker notes them as future extension).
Menu: NEW synthetic case "clienti" with "Statistiche vendite".
FATTURATO_ARTICOLI_PROD_CARDIOLINE tracked NOT_APPLICABLE (vertical of
another customer).

Verification (sandbox only): brace/paren with literal stripping,
div/table balance, SQL alias<->mapper coherence, CSV 12 cols; chain
verified with git apply --check + apply on fresh clone of HEAD bb97ba5.
Maven build and SQL Server 2012 execution to be confirmed at first
deploy.
