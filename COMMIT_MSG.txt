Add migration coverage audit: per-entry map + report

Capstone session. Complete coverage map of the legacy Aquarius menu vs
AquariusWeb, for Opus's roadmap. Version bump 0.15.0 -> 0.16.0 (rebased
on HEAD ebf1c44, which took 0.15.0 for bilancio phase 2).

METHOD: authoritative function list = tbl_menu full tree (1529 rows ->
1051 executable entries reachable from the 11 first-level menus; ~120
entries in ORPHAN groups referenced by no L1 are excluded and flagged
as likely dismissed) + user screenshots of all 19 Contabilita submenus
(131 entries classified BY HAND, entry-by-entry, with evidence) + web
coverage index (FORM_TO_URL, synthetic MenuService entries, GET
controller routes). Other areas classified by rule engine (form match
-> cited route; write/print/utility label patterns) with refinement;
residual uncertainty carried as NEEDS_DOMAIN with a precise question.

RESULT: 36 MIGRATO / 4 PARZIALE / 418 MANCANTE / 593 NON_APPLICABILE.
Counts are per menu ENTRY, not functional weight. Two large MANCANTE
blocks are actually domain questions for the user: the produzione
"film protettivi" vertical (107 entries - Impresind STANDARD-only?)
and the Consulenza area (66 entries).

DELIVERABLES
- resources/migration/menu_coverage_audit.csv: one row per menu entry
  (area, path, voce, form/prg, copertura, evidenza web, reason, note,
  classification method) - RFC-4180, 1051 rows.
- resources/migration/COVERAGE_REPORT.md: per-area and per-submenu
  matrices + 12 prioritized missing items with rationale + NEEDS_DOMAIN
  questions + open observations for Opus (CEE reconciliation,
  MOV_NREGIS vs MOV_NUMPRO).
- Tracker: +9 audit rows at group level; per-entry map lives in the
  dedicated audit CSV (declared deviation).

No functional module added; read-only audit. Applies on HEAD ebf1c44.
