Add read-only DDT module (/ddt)

New consultation module for U_BOL_TT + U_BOL_DD, web counterpart of VFP
menu_BOL000 ("Documenti di trasporto"). Depends on the orders patch
(applies after it in the chain).

- DdtHead/DdtRow read-only entities; list scoped to society + fiscal year.
- Column-name gotcha documented: U_BOL_* reuses the ORD_ prefix
  (ORD_NUMDDT/ORD_DATDDT = DDT number/date; ORD_NUMORD/ORD_DATORD on the
  same header = LINKED order refs).
- Header-rows join: TT.TAGGANCIO = DD.DAGGANCIO (legacy SQL join form).
- Delivered qty = ORD_QTAORD (summed by APPLILIB for order fulfilment);
  rows show linked invoice (MOV_NUMFAT/MOV_DATFAT).
- Status badge Fatturato/Da fatturare from ORD_DATFAT.
- MenuService FORM_TO_URL: menu_bol000 -> /ddt.
- Migration tracker: 7 rows for menu_BOL000.SCX.

Verification (sandbox only): brace balance OK, duplicate @Column empty,
fragment arity OK, CSV OK. Applies cleanly in chain on HEAD d4652ec.
Maven build and live-DB behaviour to be confirmed at first deploy.
