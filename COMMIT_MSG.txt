Unify prima nota menu into a single CRUD entry

The legacy tbl_menu exposes a "Prima nota" submenu fragmented into many items
(insert effettiva/previsionale, update, search, cancel, duplicate,
block/unblock, import), all greyed out as "(da migrare)". In the new system
these are all operations of a single registration CRUD, not separate forms.

MenuService now replaces that submenu's legacy leaves with one web entry,
"Prima nota" -> /contabilita/primanota, when the L2 container is recognised
(isPrimaNotaContainer, by label). The contabilita synthetic entries keep
Storico, Bilancio and the two Partitari plus the chart-of-accounts shortcuts;
Primanota is no longer synthetic to avoid a duplicate.

The write CRUD (insert/update/duplicate/cancel-as-storno) does not exist yet,
so the entry currently opens the read-only consultation; the write actions
will be added to the same page in the data-entry slice. The menu entry is
already the final one — the page grows, not the menu.

Supersedes the earlier menu-contabilita patch (do not apply that one). Rebased
on supplier+accounting; git apply --check clean. Menu cache needs a restart to
refresh.
