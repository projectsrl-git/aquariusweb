Fix empty account description in primanota and bilancio

The registration detail and the balance showed empty "Descrizione conto"
despite correct account codes. Cause: a trim mismatch — accountNameMap keys
are trimmed (getCode().trim()), but the lookup used the raw account code,
which is char(13) space-padded in the legacy (MOV_CONTO), so
"6061000002   " never matched "6061000002".

Fix: trim the account code on lookup in registrazione.html and in
bilancio(); storico already trimmed on input.
