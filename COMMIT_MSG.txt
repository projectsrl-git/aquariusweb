Add company parameters DEEP-analysis catalog (MENU_AZI000)

Part 1 of 2 (the viewer patch applies on top of this one). DEEP analysis
of EVERY parameter exposed by the legacy form MENU_AZI000.SCX
("Definizione dati azienda"), published as a runtime-readable catalog:
src/main/resources/parametri/parametri_aziendali_catalog.csv (UTF-8,
quote-aware, 11 stable columns) + README with schema, method and
confidence criteria.

KEY ARCHITECTURAL FINDING (the reading key for every parameter):
APPLILIB.PRG, function AQUADOCU (startup), loads every AZI_X column of
U_AZI_AN/PA/PB into a public variable PUB_X (712 verified mappings).
The ERP reads the PUB_* variables, almost never the tables - so the real
usage of a parameter lives on its PUB. Exception: print/FE libraries
read some registry columns directly for headers and XML.

Numbers:
- 845 parameter controls parsed from the form (1717 objects, 14 top
  tabs, 103 pages incl. nested pageframes); labels from checkbox
  captions or geometric label attachment; optiongroup values extracted.
- 743 columns resolved on the Impresind schema (234 AN / 249 PA /
  260 PB); 98 form columns ABSENT in this DB (other release/vertical),
  included with an explicit note.
- Catalog: 841 rows - confidence 622 ALTA / 66 MEDIA / 153 INCERTO.
  purpose = the form's own descriptive caption; how_it_works = AZI->PUB
  load chain + cited code excerpt (conditionals preferred) with
  file.function; used_in = current sources only (dated backups and
  replication/cleanup libs excluded from evidence).
- No invented interpretations: 31 possible orphans and every ambiguous
  case marked INCERTO with a note.

Migration tracker: 3 rows for MENU_AZI000.SCX (analysis PARTIAL, editing
WRITE_SIDE). Slice doc in .claude/.

Verification (sandbox only): CSV parse-back 841 rows x 11 fields, no
embedded newlines; applies cleanly on fresh clone of current HEAD.
