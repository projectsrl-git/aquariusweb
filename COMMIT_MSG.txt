Add configurable per-environment logging via logback

Replace Spring's logging.file.* with a dedicated logback-spring.xml whose
output directory is driven by the property aquarius.log.dir. Precedence:
AQUARIUS_LOG_DIR env var > -Daquarius.log.dir > per-profile value > ./logs.

Each environment gets its own directory (dev/sit/uat/prod under
D:/logs/aquariusweb/<env>), so logs are easy to find and archive. Rotation
is daily plus by size (10MB), archives gzipped under <dir>/archive/, kept
90 days with a total size cap. A separate aquarius-error.log collects
WARN/ERROR for fast triage; the console appender stays so catalina.out keeps
mirroring output.

This also makes the current post-login failure readable: with
AQUARIUSWEB_HOME=".", the previous log path resolved against Tomcat's
working directory and was effectively lost.

Verification: logback XML validated, aquarius.log.dir resolves in every
profile, git apply --check clean on a fresh clone of HEAD e67cd01. Runtime
logging under Tomcat to be confirmed on target.
